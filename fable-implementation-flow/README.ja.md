# fable-implementation-flow

[English](./README.md) | 日本語

**Fable 流の実装ワークフロー**で、機能・アプリ・ゲーム・プロトタイプ・バグ修正を構築します。ざっくりした依頼を、ブリーフから検証済みの動くコードまで一気通貫で形にします。

## 概要

このプラグインは `fable-implementation-flow` スキルをパッケージ化したものです。アドバイスだけでなく「最後まで実装してほしい」依頼や、「Fable thought」「思考プロセス」「実装ログのように」といった進め方を指定したときに、Claude が自動で起動します。

スキルは再現可能なエンジニアリングのループを定義しています。

1. **依頼をフレーミング** — プラットフォーム・フレームワーク・UI 言語・アセット・永続化・検証経路を推定する。
2. **まずローカルのコンテキストを収集** — 編集前にプロジェクトのファイル・設定・既存の慣習を読む。
3. **具体的なアーキテクチャを決める** — 状態オブジェクト・サービス・モデル・定数・分離制約を明示する。
4. **時間のかかる作業を早めに着手** — アセット/ビルドはバックグラウンドで走らせつつ実装を進める。
5. **土台から外側へ実装** — まず設定、次にモデル、コア機構、最後に仕上げ。
6. **システムをリプレイして検証** — 重要フローを辿り、数値の境界条件を確認し、ビルド/テストを実行する。
7. **短いフィードバックループで修正** — 該当 API を正確に確認し、最小限のローカル修正を優先する。
8. **エビデンスで締める** — 動作を確認し、何を変え・どう検証したかを要約する。

## インストール

### Claude Code から（推奨）

```
/plugin marketplace add nogu66/claude-code
/plugin install fable-implementation-flow@nogu66/claude-code
```

上記を実行後、Claude Code を再起動するとプラグインが有効になります。

### 手動インストール

```bash
git clone https://github.com/nogu66/claude-code.git
```

その後、Claude Code 上で:

```
/plugin marketplace add ./claude-code
```

## 使い方

タスクが説明にマッチするとスキルは自動で起動します。明示的に促すこともできます:

```
fable-implementation-flow を使って、この機能をブリーフから検証まで作って
```

## 得られるもの

- 計画だけでなく、完成して検証済みの成果物。
- 短く有用な進捗報告（生の思考過程ではなく、決定と確認）。
- 何を変え・どう検証し・残るリスクは何かを簡潔にまとめた最終要約。
